Run verifier for the current round, given a prover message.

Note that verify_round_and_update_state only samples and stores challenges; and update the verifier’s state accordingly. The actual verifications are deferred (in batch) to check_and_generate_subclaim at the last step.

Source

fn check_and_generate_subclaim( &self, asserted_sum: &F, ) -> Result<Self::SumCheckSubClaim, Error>

This function verifies the deferred checks in the interactive version of the protocol; and generate the subclaim. Returns an error if the proof failed to verify.

If the asserted sum is correct, then the multilinear polynomial evaluated at subclaim.point will be subclaim.expected_evaluation. Otherwise, it is highly unlikely that those two will be equal. Larger field size guarantees smaller soundness error.
